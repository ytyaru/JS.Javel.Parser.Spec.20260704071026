# Javel.Parser.Spec

日本語小説構文解析器仕様。

## 概要

テキストをHTMLに変換する。Markdownのような独自構文を策定してHTMLに変換するライブラリを作る。将来はHTMLを解析して本のようにページを捲って表示できるようにする。

## 入出力

* 原稿(TEXT)→本ライブラリ(JavelParser)→小説(HTML)
* 小説(HTML)→本ライブラリ(JavelParser)→原稿(TEXT)

原稿とHTMLは相互変換できること。そのために出力結果のHTMLから原稿TEXTに復元できる情報を持たせること。

## 趣旨

素人が自分で書いた書いた小説を自分で本のように読めるようにしたい。

* 多くの環境で動作すること
* 無用な危険や費用を排除すること
* すべてを自分で管理・制御できること

既存アプリやサービスでは満たせない。アプリはスマホでしか使えなかったりする。小説投稿サイトはアカウント必須であり無用な社会的リスクやコストを支払わされる。

素人が自分で書いた小説を自分で読むだけの場合、既存アプリ・サービスでは制限や危険が大きすぎる。本来ならもっとシンプルに実現できるはずだが、利益にならないため誰も作らない。よって自分で作るしかない。

既存アプリやサービスは次のような問題がある。今回の趣旨に合わないため自分で開発する必要がある。

* 本のように読めない（画面境界で文字が見切れる）
* ユーザに自由がない（提供側の営利目的に束縛される）
	* 利用できない
		* 特定プラットフォーム・デバイス・OS・バージョンを強要される
	* 継続できない
		* サービス終了や改悪を強要される
	* 社会的・法的リスク増大
		* 個人情報提供義務
			* アカウント必須
	* データ全損・漏洩リスク
		* パスワード管理・漏洩・忘却
	* 趣旨外の時間・労力消耗
		* 規約の理解遵守必須
	* ソースコードが公開されていない

## 入出力

入出力の書式を定義する。

1. 原稿
2. HTML

### 1. 原稿

1. 表紙
2. 本文

``````
表紙
本文
``````

#### 1. 表紙

``````
キャッチコピー
https://
...
タイトル
https://
...
著者名
https://
...
MONAコインアドレス

説明文(80字)。

帯文(200字)。

カテゴリ
ジャンル
タグ
注意
``````

表紙は4つの段落で構成される。一つの段落は2連続改行コードで区切る。

段落|項目
----|----
1|キャッチコピー、タイトル、著者名、コイン
2|説明文
3|帯文
4|カテゴリ、ジャンル、タグ、注意

必須項目はタイトルと著者名のみ。それ以外はすべて任意。URLもすべて任意。

URLは複数入力可能。理由は複数のストレージやSNSサービスが同一対象を示す場合があるから。キャッチコピーのURLは受賞した時などにその情報源をURLで示す。

説明文はhtmlのmeta description。Google検索時は先頭80字まで表示する。

帯文は本にある帯文を模したもの。200字まで。

カテゴリ・ジャンルは任意で大分類〜小分類などの包含関係を表現できるよう以下のように`＞`,`＜`をメタ文字にする。

```
大＞中＞小
小＜中＜大
```

カテゴリとジャンルは例えば以下のように使い分ける。書類としての分類と、内容としての分類に分ける。

```
創作文＞小説
ファンタジー＞ダーク
```

タグは全角スペース区切りとする。作品やその対象者を示すキーワード。

```
タグ１　タグ２　タグ３
```

注意は以下のいずれか。全角スペース区切り。読者に対してこのような描写があると注意する。

```
性　暴力　残酷
```

将来的にフェンス構文でYAML形式の設定データをフロントマターのようにセットする仕組みを作るかもしれない。

#### 2. 本文

1. フェンス
2. ブロック
3. インライン

原稿は大まかに上記3つの部品で構成される。それぞれ特定の構文特性を保っている。

構文解析は原稿をトークンに分解し、データ付与して意味付けしたものをノードとし、その構造を構文木(AST)とすることで行う。

##### 1. フェンス

フェンスはMarkdownのものと同じ。メタデータや本文とは異なる書式をパースする時に用いる。改行コードすらデータに含めることができる唯一の書式である。

* フェンス開始記号
	* `---`のように行頭に特定の記号を3つ以上連続して開始する（フェンス開始記号）
	* フェンス引数: フェンス開始記号の直後に続く字
* フェンス内容（開始行末端〜終了行先端の間にある全テキスト）
* フェンス終了記号: 開始記号と対になること
* 記号数を増やしてネストできること

尚、フェンスの具体的な用途は未定。但し将来の拡張に備えてフェンス要素の構文解析は実装する。

``````
---revision
- {date:'YYYY-MM-DDTHH:mm:ss', diff:''}
- {date:'YYYY-MM-DDTHH:mm:ss', diff:''}
- {date:'YYYY-MM-DDTHH:mm:ss', diff:''}
---
---parse
ruby: 
	type: 青空文庫書式／カクヨム／note
---
---split
---
---style
---
----parent
---child
---
----
``````

##### 2. ブロック

ブロックは小説構文用の最大単位。2連続改行コードでブロック分離する。ブロック要素のうち以下3種類を標準実装する。

1. 見出し: `<h1>`〜`<h6>`
2. 段落: `<p>`
3. 主題区切り: `<hr>`

###### 1. 見出し: `<h1>`〜`<h6>`

`＃`がメタ文字。

``````
＃見出し１

＃＃見出し２

＃＃＃見出し３

＃＃＃＃見出し４

＃＃＃＃＃見出し５

＃＃＃＃＃＃見出し６
``````

###### 2. 段落: `<p>`

2連続改行コードがメタ文字。

``````
本文。

１段落目。

２段落目。

2連続改行コードで意味段落になる。
尚、段落内で1つ改行コードを入れるとbr要素が入り形式段落も表現できる。
``````

###### 3. 主題区切り: `<hr>`

`＊＊＊`をメタ文字とする。

``````
場面Ａの本文。

＊＊＊

場面Ｂの本文。
``````

##### 3. インライン

ブロック要素内にあるものをインライン要素とする。インライン要素は必ずブロック内にあり、かつ改行コードを含まない。尚、改行インライン要素は例外。

1. 改行: `<br>`
2. ルビ: `<ruby>`
3. 傍点: `<em>`
4. 割注: `<span class="warichu">`

###### 1. 改行: `<br>`

``````
段落内で
一つ改行コードを入れると
br要素が入る。
``````

###### 2. ルビ: `<ruby>`

``````
｜太郎《たろう》
``````

パイプ`｜`は省略可能。`《`の前にある全漢字を親文字とする。

``````
太郎《たろう》
``````

ルビの配置は以下3パターンある。通常は上面ルビだ。

* 上面ルビ
* 下面ルビ
* 両面ルビ

下面ルビは以下。

``````
太郎《｜たろう》
``````

両面ルビは以下。

``````
太郎《たろう｜タロー》
``````

ルビを振る単位は以下3パターンある。通常はグループルビだ。

* グループルビ
* 熟語ルビ
* モノルビ

熟語ルビは以下。パイプ必須。全角スペース`　`がメタ文字。親文字とルビ文字の両者には同数だけ全角スペースが必要。`　`で分割された各要素は必ず1字以上であること。

```
｜山田　太郎《やまだ　たろう》
```

モノルビは以下。パイプ任意。ルビ文字内の`、`がメタ文字。親文字-1個だけ`、`が必要。`、`で分割された各要素は必ず1字以上であること。

```
｜山田太郎《やま、だ、た、ろう》
```

ルビの配置と単位を組み合わせることも可能。

```
｜山田太郎《やま、だ、た、ろう｜タロー》
｜山田　太郎《やまだ　たろう｜タロー》
```
```html
<ruby style="ruby-position:under">
  <ruby style="ruby-position:over">
    山田
    <rt>やまだ</rt>
  </ruby>
  <ruby style="ruby-position:over">
    太郎
    <rt>たろう</rt>
  </ruby>
  <rt>タロー</rt>
</ruby>
```

###### 3. 傍点: `<em>`

``````
＊傍点＊
``````

`＊`がメタ文字。開始と終了の範囲を`＊`で囲むこと。

傍点のCSSは`﹅`記号を使うのがデフォルトとする。但し設定で変更可能にしたい。その場合はフェンス要素で設定させることになるだろう。詳細未定。

メタ文字の重複とエスケープについて。ブロック要素のメタ文字としては主題区切りを意味する`＊＊＊`があるし、伏字としてプレーンテキストで`＊＊＊`のように表記したい場合もある。これらと傍点を区別できるようにすべきだ。

まず傍点の場合、対象文字に`＊`があってはならないこと。また、対象文字は１字以上存在すること。それをルールにする。これで伏字のように連続したアスタリスクは傍点の対象外にできる。例えば`＊＊`や`＊＊＊`はプレーンテキストとして出力される。また、行内に`＊`が一個しかない場合も同様。

しかし行内に`＊`をプレーンテキストとして出力したい箇所が二箇所ある時、その間を対象とした傍点構文として解釈されてしまう。そこで明示的にメタ文字がプレーンテキストであることを指示するために`\＊`という記述が必要だ。`\＊アスタリスクで囲まれたテキストを出力する\＊`。のような時に使う。

`山＊太＊は＊した`のようにして、いくつかの文字が読み取れないことを表現する時に`＊`という字を使いたい場合もあるかもしれない。この時、一つずつ`\`を付与するのは面倒だから`` `山＊太＊は＊した` ``のように囲んでエスケープしたいことがあるかもしれない。

同様に、その構文が複数の段落に跨る場合もありえるかもしれない。以下のようにインライン構文だけをエスケープした専用フェンス構文を用意するか？　じつに馬鹿げている。

````
```escape inline thema-break
山＊太＊は＊した。

山＊太＊は＊した。

＊＊＊
```
````

`﹅傍点﹅`にするか？　だが入力が面倒だ。`﹅`の文字は「ごま」や「ぼうてん」で入力変換すると出てくる。アスタリスク`＊`のほうが素早く入力できる。

`《《傍点》》`にするか？　だがメタ文字の数が多すぎる。だがカクヨムなど傍点が使える小説投稿サイトは基本的にルビ記法の代用かこの記法のどちらかだ。汎用性はある。メタ文字の字種も減らせる。エスケープする時は`｜｜`,`｜《`,`｜》`のようにするらしい。でも`《《傍点》》`はエスケープできない。`｜《《傍点》》`でも`｜《｜《傍点｜》｜》`でもダメ。

1. プレーンテキストで自由に使いたい
2. 主題区切り、傍点などの素早い入力として使いたい

###### 4. 割注: `<span class="warichu">`

```
〔ここの内容が割り注になる〕
```

`〔〕`がメタ文字。割注は本文の半分サイズで表示され二行に分割して表示する。

1. 字種の幅を考慮した上での字数が偶数なら丁度半分の箇所で分割する
2. 字種の幅を考慮した上での字数が奇数なら
	1. 単語分割Intl.Segmenterの位置と合致するならそこで分割する
	2. しないなら１行目を１字長めにして分割する

通常、割注は前にある用語などの詳細解説であるため割注内を`()`など何かしらの囲み文字で囲って表示することがある。だが、囲わずに文字の小ささだけで割注であることを示す場合もある。この時、HTML表示時の囲み文字を表示するか否か、表示するならどの文字を使うかを設定可能にしたい。

デフォルト時はそのまま`〔〕`にするか？　それとも`()`にするか？　未定。この設定はフェンス要素で設定させるのが最適だと思われる。

```html
<span class="warichu">
  <span class="">(</span>
  <span class="">一行目</span>
  <span class="">二行目</span>
  <span class="">)</span>
</span>
```

###### 5. エスケープ

使用するメタ文字をそのままテキストとして出力したい場合がありえる。例えば行頭に`＃`を出力したい時など。そのままだと見出しのメタ文字と判定されテキストとして`＃`が表示されない。

他にも`｜`、`《》`、`〔〕`、`＊`といったメタ文字がある。これらを構文解析させずそのままのテキストとして表示するにはエスケープする必要がある。

もし一字で済むなら直前にバックスラッシュを付与する。`\＊`のように。もし改行を含まないインライン要素ならエスケープ用バッククォートで囲む。` ｜漢字《かんじ》 `。`` ` ｜漢字《かんじ》 ` ``。

見出しの`＃`をエスケープしたいなら`\＃`とするか、`` `＃＃＃＃＃＃` ``とする。

フェンスのエスケープはフェンス引数`escape`で行う。

`＊`について。ブロック要素としては`＊＊＊`があるし、伏字としてもテキストとして`＊＊＊`のように記述したい場合もある。これらと傍点を区別できるようにすべきだ。

まず傍点の場合、対象文字に`＊`があってはならないこと。また、対象文字は１字以上存在すること。それをルールにする。これで伏字のように連続したアスタリスクは傍点の対象外にできる。

もし`＊＊＊`というテキストしかない段落を作りたい時は？　`＊＊＊`の後に二連続改行すると主題区切りとして判定され`<hr>`要素に置換されてしまう。これを回避するにはエスケープする必要がある。

もしブロック要素である主題区切り`＊＊＊`

尚、ブロック要素としては`＊＊＊`があるし、伏字としてもテキストとして`＊＊＊`のように記述したい場合がある。もしテキストとして`＊＊＊`と表示する段落を作りたい時はどうする？　このままでは主題区切りとして識別されてしまう。またはインライン内で`＊＊＊`とすると`＊`に傍点`﹅`が付与した`<em>＊</em>`というHTMLになってしまう。

このように、メタ文字として解釈するか、出力テキストとして解釈するか、制御したい場面が出てくる。
