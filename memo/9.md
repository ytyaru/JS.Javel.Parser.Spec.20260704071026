    1. トークナイザ: Tokenizer

3. レンダラ: Renderer(AST->HTML)
    1. 表紙
    2. 目次(TOC)
    3. 本文

パーサは原稿から構文木(AST)を作る。レイアウタは画面サイズに応じて一ページあたりの要素や文字をどこまで含めるか計算してHTML要素分割する。コンバータはASTをHTMLテキストに変換する。

将来はHTML→AST→原稿に変換するクラスも実装したい。

APIは、原稿が10万字以上でもフリーズしないようにしたい。コンバータのAPIはasyncなメソッドを用意してawaitでwaitを入れることでフリーズを防げるようにしたい。パーサ／レイアウトのAPIはgeneratorなメソッドを用意してASTのNode単位／ページ単位でyieldするようにしたい。

スタイラはページ分割するために必要な基礎。画面サイズはいくつで、文字数や一行あたりの字数はいくつで、フォントファミリは何で、などのページ計算に必要なスタイル情報。これを日本語小説が読みやすい範囲内の設定値を用意しておく。また、ユーザは設定で変更できるようにする。他にも段組み数やダーク・ライトモード、書字方向などの情報もここで管理する。

スプリッタは２種類の実現方法がある。ブラウザでダミーDOMを作る実測と、フォントなどの実体を考慮しないまま仮想値として計測する方法だ。前者の場合はwindow.documentが存在する環境でなければ実行できない。だが後者はそれが存在しないBunやWebAPIで実行できる。

## 1. パーサ

まずはパーサを作る。原稿を受け取りASTを返却する。

```javascript
class Parser {
    constructor(options) {this._={options:options, t:new Tokenizer(options), l:new Lexer(options)}}
    async parse(manuscript) {
        const nodes = [];
        for (let token of this.t.tokenize(manuscript)) {
            nodes.push(this.l.lex(token));
        }
        // nodesの順序整列(構文解析の都合上Fenceのほうを先に解析する必要があったが原稿に記述された順に戻す)
        // フェンス要素だけパースする（内部のテキストを固有の構文解析で）
        // ブロック要素だけパースする（内部のインライン要素）
        yield nodes;
    }
}
```
