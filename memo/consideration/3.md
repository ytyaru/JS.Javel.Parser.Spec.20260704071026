## 「Bridgy」と「Bridgy Fed」の違い

## プロトタイプ開発：fedon 拡張機能の最小構成
それでは、現実味を帯びてきたこのツールを形にするため、「Webサイト側のJSから呼び出され、拡張機能がそれを受け取ってポップアップ（承認UI）を起動する」という、中核となる最小構成のコード（Manifest V3）を作成しました。

## 1. manifest.json (拡張機能の設定ファイル)
拡張機能の権限や、ページにスクリプトをインジェクションする設定を記述します。

  "name": "Fedon - Decentralized Web Commentary",
  "version": "1.0.0",
  "description": "FediverseとWebサイトを繋ぐブラウザ拡張機能",
  "permissions": [
    "storage"
  ],
  "background": {
    "service_worker": "background.js"
  },

    {
      "matches": ["<all_urls>"],
      "js": ["content.js"],
      "run_at": "document_start"
    }
  ]
}

## 2. content.js (Webサイト側に window.fedon を注入するスクリプト)
Webサイトの通常のJSからアクセスできるグローバルオブジェクト window.fedon を作成し、呼び出されたら拡張機能の本体（Background）へメッセージを転送します。

// ページ側に安全にAPIオブジェクトを注入する（Isolated WorldからMain Worldへの注入）const injectScript = `
  window.fedon = {
    getUsers: async () => { return await window.fedon._send('getUsers'); },
    activate: async (info) => { return await window.fedon._send('activate', { info }); },
    post: async (text) => { return await window.fedon._send('post', { text }); },
    like: async (url) => { return await window.fedon._send('like', { url }); },
    reply: async (url, text) => { return await window.fedon._send('reply', { url, text }); },
    
    bridgyFed: {
      like: async (url) => { return await window.fedon._send('bridgyFedLike', { url }); },
      reply: async (url, text) => { return await window.fedon._send('bridgyFedReply', { url, text }); }
    },

    // 拡張機能の本体へデータを送る内部関数
    _send: (action, data = {}) => {
      return new Promise((resolve) => {
        const eventId = Math.random().toString(36).substring(2);
        // カスタムイベントを使ってcontent.js経由でbackgroundへ中継する
        window.dispatchEvent(new CustomEvent('fedon-request', {
          detail: { eventId, action, data }
        }));

        // レスポンスを待つ
        window.addEventListener(\`fedon-response-\${eventId}\`, (e) => {
          resolve(e.detail);
        }, { once: true });
      });
    }
  };
`;
// スクリプトタグとしてHTMLに埋め込むconst script = document.createElement('script');
script.textContent = injectScript;
(document.head || document.documentElement).appendChild(script);
script.remove();
// Webページからのリクエストイベントをキャッチして、Background(Service Worker)に送る
window.addEventListener('fedon-request', (event) => {
  const { eventId, action, data } = event.detail;
  
  chrome.runtime.sendMessage({ action, data }, (response) => {
    // 処理結果をWebページ側に返す
    window.dispatchEvent(new CustomEvent(`fedon-response-${eventId}`, {
      detail: response
    }));
  });
