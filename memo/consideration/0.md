# 小説体系Javel

原稿からページ捲りで閲覧できるHTMLを生成する。

* 体系 Javel = Japan + Novel
* 構文 **: Javel Syntax
* 原稿 jv: Javel Manuscript
* 様式 jl: Javelay = Javelook Layout
* 履歴 jr: Javion = Javel Revision
* 仮本 jk: Javelook = Javel + Book + Look

各中間ファイル形式の拡張子を定義する。原稿ファイルの拡張子は`jv`とする。

`jv`と`jl`の二つを併せて`jk`を作る。`jk`は特定の画面サイズでページ捲りで閲覧できるHTMLファイル。中にCSSやJSがある。

`jv`と`jl`を一つのファイル形式にしたものが`jvl`。原稿は恐らく変わらないが、様式は環境次第で変わる。それをわざわざ一つにまとめるのは著者が確認した環境として記録・公開するため。

もし原稿`jv`をDeflate圧縮したら`jvd`、もし`jv`や`jk`をGZip圧縮したなら`jvz`,`jkz`、もし`jk`に挿絵など他のファイルをTarball化してGZip化したなら`jktz`。

将来、公開用データ構造として履歴すなわちリビジョン管理メタデータも含めるかもしれない。この時その規格を`jr: Javel Manuscript Revision`とする。日時とdiffを一つのセットとしたリスト。リビジョン数はそのままリスト数で示す。

原稿・履歴を併せ持つ形式を`jvr`、原稿・様式・履歴を併せ持つ形式を`jvlr`とする。

EPub、Markdown、各小説投稿サイトとの相互変換もしたい。

PDF化し印刷可能にして製本可能にしたい。

# 小説構文Javel

小説用の構文を定義する。日本語入力に最適化した。名前は`Japan + Novel = Javel`で`Javel`と仮称する。

Javelは楽に入力できるよう、なるだけ全角文字だけを使う。メタ文字も全角を使う。但し詳細入力には半角文字も使うこととする。

## 基本仕様

* ファイル形式: テキスト
* 文字セット: UTF8
* 拡張子: `txt`, `javel`, `jv`
* データ量: 文庫本一冊分（10万字程度）を想定

## 書式

Markdownのような書式にする。

大まかに言えば表紙と本文の二つに大別する。最初の見出しより前が表紙、後が本文である。

### 表紙

表紙は`＊＊＊`の後に裏表紙を書くことができる。書かないこともできる。

```javel
作品題名
著者名

＊＊＊

完

読了ありがとうございました。感想の投稿など読了後もお楽しみ頂けると幸いです。
```
